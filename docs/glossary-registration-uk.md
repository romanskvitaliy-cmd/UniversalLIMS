# Словник термінів — реєстратура (UniversalLIMS)

> Для реєстраторів і розробників. Оновлено: 2026-05-30.

## Сутності в системі

| Термін у UI | Що це | На папері часто називають |
|-------------|--------|---------------------------|
| **Справa / замовлення** | Один візит клієнта, один `Order` | — |
| **Номер справи (REF-…)** | Автоматичний номер при створенні справи | «Номер направлення» у журналі |
| **Замовник** | Клієнт (`Customer`) | — |
| **Проба** | Один зразок / одне дослідження зі своїм номером (`Sample`) | Проба, зразок |
| **Бланк PDF** | Обраний шаблон для заповнення (`OrderDocument`) | Часто «протокол» (Ф327, 205/о, 332…) |
| **Направлення (бланк REF-*)** | PDF-бланк направлення на дослідження (`OrderDocument`, шаблон REF-*) | Бланк направлення до лабораторії |

## Правила на формі «Прийом справи»

1. **Один рядок** = **одна проба** + **бланк REF-*** + **бланк протоколу** (якщо REF-шаблони є в системі).
2. Кілька рядків = кілька проб (різні точки відбору або різні бланки).
3. **Номер REF-…** — **номер справи** в LIMS, не паперове направлення на пробу.
4. Бланк **REF-*** — направлення на дослідження; на пілоті **1 на пробу** (roadmap §7.0). Режим «1 REF на всю справу» — backlog.
5. На PDF напис «ПРОТОКОЛ» — це **назва форми**, у системі в рядку обираєте **бланк для проби**.

## Мапінг полів

Якщо в справі **2+ бланки** (проби), можна об’єднати спільні поля (ПІБ, дата відбору…) — один ввід потрапляє на всі обрані PDF.
